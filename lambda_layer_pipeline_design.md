# Lambda Layer 自動ビルド・デプロイ パイプライン 設計概要

---

## 1. 全体像

本設計は、**外部ライブラリ管理用パイプライン**と**Lambdaレイヤーデプロイ用パイプライン**の2段構成により、Lambda Layer の資材を自動的にビルド・管理・デプロイするものです。

複数の対象ライブラリ・レイヤーが存在する場合も、同一の仕組みで対応できるよう拡張性を考慮しています。

---

## 2. パイプライン構成

### 2-1. 外部ライブラリ管理パイプライン（CodeBuild 単体）

外部ライブラリの取得・ZIP化・S3格納を行うパイプラインです。  
定期実行や手動トリガーにより、最新ライブラリを常に S3 に保持します。

```
[トリガー: 手動 / スケジュール実行]
        │
        ▼
┌─────────────────────────────────────────────┐
│              CodeBuild                      │
│                                             │
│  ① requirements.txt を読み込む              │
│  ② 各行のライブラリを pip install でDL      │
│  ③ ダウンロードした資材を ZIP 化            │
│  ④ 指定の S3 バケット/プレフィックスへ格納  │
└─────────────────────────────────────────────┘
        │
        ▼
  [S3: 外部ライブラリ保管バケット]
  例) s3://my-bucket/layers/{layer名}/external_libs.zip
```

**処理詳細:**

| ステップ | 内容 |
|---|---|
| ① 読み込み | `requirements.txt` の各行からライブラリ名・バージョンを取得 |
| ② DL | `pip install -r requirements.txt -t ./python/` でローカルへインストール |
| ③ ZIP化 | `./python/` ディレクトリをZIPアーカイブ化 |
| ④ S3格納 | `aws s3 cp` で指定プレフィックスへアップロード |

---

### 2-2. Lambda Layer デプロイパイプライン（CodePipeline + CodeBuild）

GitHub上の自作ライブラリと S3 の外部ライブラリを合算し、Lambda Layer 用 ZIP を生成・格納するパイプラインです。

```
[トリガー: GitHub へのプッシュ]
        │
        ▼
┌─────────────────────────────────────────────┐
│  CodePipeline                               │
│                                             │
│  ┌──────────────────────────────────────┐   │
│  │ ステージ①: ソース取得                │   │
│  │  GitHub リポジトリから最新コードを取得 │   │
│  └──────────────────────────────────────┘   │
│                │                            │
│                ▼                            │
│  ┌──────────────────────────────────────┐   │
│  │ ステージ②: ビルド / デプロイ         │   │
│  │  CodeBuild                           │   │
│  │                                      │   │
│  │  ① S3から外部ライブラリZIPをDL       │   │
│  │  ② 外部ライブラリを展開              │   │
│  │  ③ GitHubからの自作ライブラリと合算  │   │
│  │  ④ 合算資材をZIP化                  │   │
│  │  ⑤ Layer格納用S3バケットへアップロード│   │
│  └──────────────────────────────────────┘   │
└─────────────────────────────────────────────┘
        │
        ▼
  [S3: Layer格納用バケット]
  例) s3://my-bucket/layers/{layer名}/layer_latest.zip
```

**処理詳細（ステージ②: CodeBuild）:**

| ステップ | 内容 |
|---|---|
| ① 外部ライブラリDL | S3から `external_libs.zip` をダウンロード |
| ② 差分確認 | S3 の外部ライブラリと現在のバージョンを比較（最新か確認） |
| ③ 資材合算 | 外部ライブラリ + GitHub の自作ライブラリを同一ディレクトリへ展開 |
| ④ ZIP化 | Layer 規約に沿ったディレクトリ構造（`python/`）でZIPアーカイブ化 |
| ⑤ S3格納 | Layer格納用バケットの指定プレフィックスへアップロード |

---

## 3. S3 バケット構成

| バケット種別 | 用途 | 格納パス例 |
|---|---|---|
| 外部ライブラリ保管バケット | pip でDLした外部ライブラリ | `s3://libs-bucket/layers/{layer名}/external_libs.zip` |
| Layer格納用バケット | Lambda Layer登録用の最終ZIP | `s3://layer-bucket/layers/{layer名}/layer_latest.zip` |

## 4. 複数対象への対応

ライブラリ・レイヤーが複数存在する場合、以下のいずれかの方式で対応します。

### 方式A: パラメータ化（推奨）

CodeBuild の環境変数または `buildspec.yml` のパラメータとして `LAYER_NAME` を渡し、単一のパイプライン定義で複数レイヤーを処理します。

```
CodePipeline (layer-A 用)  ──→  CodeBuild（LAYER_NAME=layer-A）
CodePipeline (layer-B 用)  ──→  CodeBuild（LAYER_NAME=layer-B）
```

- パイプライン定義は共通化、対象レイヤーごとにパラメータのみ変更
- S3 パスも `{layer名}` で動的に切り替え

### 方式B: 並列ステージ化

1つの CodePipeline 内に複数のビルドアクションを並列配置し、一括処理します。

```
[ソース取得]
     │
     ├──→ CodeBuild（layer-A）
     └──→ CodeBuild（layer-B）
```

- 全レイヤーを同時にビルドしたい場合に有効
- パイプラインの変更コストが増加するため、レイヤー数が少ない場合に適する

## 5. 資材フロー全体図

```
GitHub リポジトリ
  │  (自作ライブラリ)
  │
  ▼
[CodePipeline: ソースステージ]
  │
  ├─────────────────────────────────────────────────────┐
  │                                                     │
  │                                          [CodeBuild: 外部ライブラリ管理]
  │                                            requirements.txt → pip install
  │                                                        │
  │                                                        ▼
  │                                          [S3: 外部ライブラリ保管バケット]
  │                                                        │
  ▼                                                        │
[CodePipeline: ビルド/デプロイステージ]  ←────────────────┘
  │  (外部ライブラリDL + 自作ライブラリ合算 + ZIP化)
  │
  ▼
[S3: Layer格納用バケット]
  │
  ▼
[Lambda Layer 登録]（手動 or 自動トリガー）
```

## 6. 補足・考慮事項

| 項目 | 内容 |
|---|---|
| 差分管理 | S3 上のライブラリZIPのハッシュ値や更新日時を比較し、変更がない場合はビルドをスキップ可能 |
| バージョン管理 | S3 のバージョニング機能を有効化し、過去バージョンへのロールバックに対応 |
| Layer の ZIP 構造 | Python の場合、`python/` ディレクトリ直下にライブラリを配置する必要がある |
| セキュリティ | CodeBuild の IAM ロールに S3 の読み書き権限のみ付与し、最小権限を徹底 |
| 通知 | CodePipeline の失敗時に SNS / Slack 通知を設定し、障害を早期検知 |
